Issen · Five-Source Forensic Architecture

KNOWLEDGE → INGEST → NAVIGATE → PARSE → ORCHESTRATE

Five Navigation Primitives

  • • [P] name → inode → block   (disk/EWF/NTFS/ext4)
  • • [M] PID → EPROCESS → VA → PA  (memory)
  • • [L] timestamp → record → field  (log stream)
  • • [Q] (endpoint, query, cursor) → rows  (live)
  • • [C] hash → blob → content graph  (CAS)

Why Live Query [Q]?

  • • Data produced by query — not pre-stored
  • • Captures ephemeral state attacker cannot retroactively destroy
  • • Cross-artifact correlation in a single API call
  • • Query itself is part of the evidence chain
  • • Crates: issen-remote-access, issen-parser-velociraptor

Why Content-Addressed [C]?

  • • Identity = hash: globally addressable, tamper-evident
  • • Merkle DAG: graph navigation, not linear seek
  • • Supply chain pivot: hash → systems → time → origin
  • • Immutability guaranteed by construction, not policy
  • • Crates: cas-forensic, git-forensic, sigstore-forensic
